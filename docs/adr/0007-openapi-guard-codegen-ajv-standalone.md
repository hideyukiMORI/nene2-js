# ADR 0007 — OpenAPI Guard Codegen (Ajv Standalone)

**Status:** Accepted  
**Date:** 2026-05-22  
**Issue:** [#86](https://github.com/hideyukiMORI/nene2-js/issues/86)

---

## Context

[ADR 0006](0006-openapi-codegen-types-only.md) committed **types-only** codegen (`openapi-typescript` → `src/generated/openapi.ts`). Runtime guards (`isHealthResponse`, `isExampleNoteResponse`, …) remain hand-written in `src/types/*` and can drift when `contracts/openapi.yaml` changes.

[#86](https://github.com/hideyukiMORI/nene2-js/issues/86) Phase A evaluated tools to generate guards from the same pinned OpenAPI input.

## Decision

1. **Tool:** [Ajv 8](https://ajv.js.org/) with **standalone code generation** (`ajv/dist/standalone`) — validators compiled at codegen time, **no Ajv runtime dependency** in the published npm package.
2. **Input:** dereferenced `contracts/openapi.yaml` via `@apidevtools/swagger-parser` (same pin as ADR 0002).
3. **Output (Phase B):** committed `src/generated/guards.ts` regenerated by `npm run codegen:guards`; drift gate `npm run codegen:guards:check` in `npm run check`.
4. **Schema set:** `components/schemas` used by `createNene2Client` responses/requests (11 schemas — see `GUARD_SCHEMA_MAP` in `tools/codegen-guards-spike.mjs`).
5. **Public API:** keep existing guard **function names**; hand modules become thin wrappers importing generated validators with TypeScript `value is T` predicates (Phase C).
6. **Format handling:** strip OpenAPI `format` (e.g. `int64`, `uri`) before compile — standalone ESM output otherwise `require()`s `ajv-formats` (incompatible with `"type": "module"`). Integer/string checks match current hand guards.
7. **Hand-maintained exceptions** (not generated):
   - `src/problem/guards.ts` — RFC 9457 subset + NENE2 problem type URIs
   - `ListNotesParams` / `ListTagsParams`, `SmokeCheckResult` — not named OpenAPI schemas
8. **Phase A spike:** `tools/codegen-guards-spike.mjs` + `tests/codegen/guards-spike.test.ts` prove parity with hand guards on JSON fixtures (output not committed until Phase B).

## Consequences

### Positive

- Contract bump → regenerate types **and** guards in one PR workflow
- Published bundle stays free of Ajv / Zod runtime deps
- Spike tests lock generated vs hand guard parity on fixtures

### Negative / trade-offs

- Generated guard file is large and machine-readable (like `openapi.ts`); excluded from strict ESLint
- `format` stripping may accept values outside strict int64/uri ranges (same as hand guards today)
- `ProblemDetails` guards stay duplicated until a separate ADR extends generation rules

## Alternatives considered

| Option                       | Verdict                                                                                                     |
| ---------------------------- | ----------------------------------------------------------------------------------------------------------- |
| **Zod / openapi-zod-client** | Rejected for publish path: Zod runtime in consumer bundle unless codegen emits plain TS (not available).    |
| **typia**                    | Rejected: TS transform plugin; conflicts with current `tsc` build and AI-readable simplicity.               |
| **Keep hand guards forever** | Rejected: drift risk; blocks 1.0.0 criterion 4 ([#84](https://github.com/hideyukiMORI/nene2-js/issues/84)). |
| **Ajv at runtime**           | Rejected: adds dependency and compile cost on every validation in consumer apps.                            |
| **openapi-generator client** | Rejected: ADR 0001/0006 — no full client codegen.                                                           |
