## P1: ミラー opt-out フラグ実装（Issue #119 / PR #120）

- **触った物**: `mirrorAuthorizationHeader?: boolean` を `Nene2TransportConfig`・`Nene2ClientConfig` の両方に追加。transport は `resolveTransportConfig`（既定 `true`）→ `send()` → `buildTransportHeaders` でミラー set をゲート、client は `resolveConfig` → `buildAuthHeaders`。`tests/transport/mirror.test.ts` に opt-out の describe を追加。
- **裏取り**: 既定 `true` = 現行挙動不変（＝semver minor・非破壊）、`false` で `Authorization` のみ送信、を全 transport path＋typed client でテスト断言〔実測〕。ローカル gate（type-check / lint / contracts / codegen guards / build / test **599** / pack:smoke）緑〔実測〕。mirror テストは 15 → **18** 本〔実測〕。
- **★教訓**: 版名の先出しは事故クラス。docs の「available since 1.3.0」は hub 裁定で「the next minor release」表記へ、CHANGELOG は `[Unreleased]` のまま版番号を除去した（未 publish の版番号を公開 docs で主張しない）。version 確定はリリース commit の手番。
- **#119 相乗り**: 残っていた固有ホスティング名を `src/transport/headers.ts` / `src/client/request.ts` / `tests/transport/mirror.test.ts` の3コメントから一般表現へ除去（同ファイルを触る P1 に同梱＝最小コスト、との起票時 disposition どおり）〔実測〕。
